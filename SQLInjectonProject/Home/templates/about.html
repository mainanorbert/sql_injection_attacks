About

The intention of this project is to fulfill a partial requirement for a Master’s degree in Cyber Security by demonstrating a deep understanding of SQL injection vulnerabilities and defenses. The project aims to analyze and contrast insecure and secure coding practices within web applications, using practical examples to highlight the risks and countermeasures associated with SQL injection attacks. This research contributes to the field by providing insights into effective security measures, particularly focusing on the implementation of parameterized queries and ORM frameworks, thereby advancing the understanding and practice of secure software development.

Our Objective

The primary goal is to offer a controlled environment where users can learn about SQL injections in a practical setting. The project is divided into two levels:
Low: This level is intentionally designed with various vulnerabilities to allow users to practice different SQL injection techniques. By interacting with this level, users can understand how SQL injections exploit weaknesses in the system.
High: In contrast, this level is engineered to be secure against SQL injection attacks. It demonstrates best practices for preventing such vulnerabilities and reinforces the importance of robust security measures.
Through this approach, users will gain a comprehensive understanding of SQL injection attacks and the importance of secure coding practices. Please remember that this site is intended solely for educational purposes and should not be used for any malicious activities.
